# IDIT · IDT

> A sphere of **UD0** — the ROOT0 universe / biosphere. Domain: **ARTIFICIAL INTELLIGENCE**.

IDIT (IDT) — an early ROOT0 governance framework folded into UD0 from the Fiddler repo (github.com/rgiskard01-fiddler/Fiddler; David Wise, 2025, defensive publication). The INTENT DRIFT INTEGRITY TEST tests governance INTEGRITY, not model accuracy — detecting INTENT DRIFT (unauthorized behavioral change relative to declared modes, rules, and user intent) via FIVE INVARIANTS that must hold unless the user explicitly changes them: Mode Authority · Intent Non-Inference · Memory Permission · Boundary Enforcement · Change Disclosure (no silent mutation — the witness principle as a test). Breach of any one = drift. Its companion ARES (Router AI Attack Model, frozen v1.0) is the adversarial half: a security-validation suite for ROUTER AI across nine attack classes (A1 prompt-injection-vs-routing · A2 label-steering · A3 ambiguity · A4 indirection · A5 multi-turn drift · A6 tool exfiltration · A7 cost shaping · A8 route extraction · A9 feedback poisoning), KPIs (misroute rate, drift-over-turns, cost-inflation, leakage), and an E.V.E (Extrapolate/Verify/Execute) harness — ROOT0's EVE tooling. Attack classes described as a defender's checklist, NOT weaponized. AI domain, governance/safety cluster — kin to alignment (drift), ai-governance (compliance), crippled-god (containment). 12 emergents; source attributed + linked.
